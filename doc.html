How to make a coffee API Documentation

Overview

The documentation covers a simple business scenario that how to make a coffee API.

Media Type support

All server response bodies and request bodies MUST be valid JSON Media Type messages.

Connection between Order and Payment

An order and a payment share a common ID.

URLs and Operations

Below are the URLs and the operations associated with them.

The URL(/coffee/) is the base URL for creating, getting, amending, cancelling orders and creating, getting payments. The following tables show all the related operations, their URLs and method.

Input

create order input

{
    "type_coffee" : "latte"
    }

get single order or get single open order input: add the order_id at the of the URL


    "/coffee/get/order/1"
    "/coffee/get/openorder/1"

update order input

{
    "type_coffee" : latte",
    "additions"" : [
    "milk",
    "shot"
    ]
    }

pick up order input: add the order_id at the of the URL


    "/coffee/pickup/order/1"

release order input: add the order_id at the of the URL


    "/coffee/release/order/1"

delete order input: add the order_id at the of the URL


    "/coffee/delete/order/1"

card payment input: It is expected to add the order_id at the end of the URL

The card's expires date should be greater than current date. Otherwise the order can not been paid and returns a error message.
"/coffee/payment/order/1" 
{
    "cardNo" : 43243,
    "expires" : "2018-03-23",
    "name" : "ming",
    "type_payment" : "card"
    }

cash payment input: It is expected to add the order_id at the end of the URL

"/coffee/payment/order/1" 
{
    "type_payment" : "cash"
    }

get specific payment input: add the order_id at the of the URL


    "/coffee/get/payment/1"

Error Return

When client dose not enter correct JSON type or JSON message, an error return looks like this:


    "Data is not valid"

We have our own menu which includes some coffee and the price of the specific coffee. If client orders the coffee that is not in the mene. A menu return looks like this:

[
    "sorry, we do not have what you order. The following is our menu."
    {
    Affogato : 5,
    Cappuccino : 4,
    Double Espresso : 6,
    Espresso : 4,
    Flat White : 7,
    Long Black" : 6,
    Long Macchiato : 6,
    Mocha : 5,
    Piccolo Latte : 5,
    Ristretto : 5,
    Short Macchiato : 7,
    latte : 3

    ]

There are many situations that can return a false message, like:the order has been paid, you can not amend the order or The order dose not exist. Errors returns samples look like these:


    "You can not amend the order detail."


    "You can only pay the payment by cash or card or You have paid for the order."


    "Please fill out the card info and using correct type."


    "Your card has expired."


    "The order has been paid or the order dose not exist."
